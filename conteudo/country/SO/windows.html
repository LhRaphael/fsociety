#Sistemas Operacionais[Windows]

$Entendendo um pouco melhor

Windows é o sistema para desktops domésticos mais usado no mundo, desenvolvido pela gigante microsoft, que despensa apresentações. Ele atualmente está na versão 11 e passou por diversas modificações no decorrer de sua história.

A princípio o sistema era baseado na linguagem C e, bem no início, era comercializado em disquetes, o que tornava os processos de instalação longos e sequenciais

Uma das razões pelas quais o Windows é amplamente utilizado hoje, além do marketing da Microsoft, é pela simplicidade dele se comparado a sistemas baseados em Linux, coisa que ocorre pelo fato do sistema ser um software completamente privado e, portanto, muito mais simples de ser gerenciado.

$Descendo as camadas

Como nos já discutimos antes, um sistema operacional é um conjunto de softwares que tratam de coordenar os recursos de hardware, mas se você tem um pouco mais de 12 anos acredito que já saiba que não existe truque de mágica nessa história, então vamos entender melhor como o windows atua.

A começar pela divisão dos diretórios, independente de qual sistema operacional alguem use, todo sistema está dividido em pastas, essas que partem de um ponto específico dentro do hardware de armazenamento, chamamos esse ponto de ponto de montagem. O ponto de montagem padrão do windows é, normalmente, o diretório C:\, e todas as outras pastas e arquivos são apenas 'galhos' desse ponto.

A partir do ponto de montagem são definidas as pastas essenciais do sistema. No caso do windows elas são: Windows, Program Files, Users, ProgramData e o subdiretório da pasta Windows, System32. Cada um possui arquivos essenciais para o funcionamento dos sitema, com enfase no System32, mas vamos entender melhor esses diretórios.

$Pastas do windows

C:\Windows\
Aqui é onde estão guardados os principais elementos para o funcionamento do sistema como o próprio kernel, bibliotecas, arquivos temporarios, drivers e os arquivos de configuração como fontes e pacotes de idioma.
C:\Program Files\
Aqui é onde os programas usados pelo usuário são armazenados, assim como as dependências próprias de cada programa.
C:\Users\
Armazena as pastas de usuários individuais, onde cada conta de usuário tem sua própria subpasta que inclui: Desktop(Área de trabalho do usuário), Documents(Documentos pessoais), Downloads(Arquivos baixados), AppData(Pasta oculta que contém dados de aplicativos e configurações de programas)
C:\Temp\
Pasta temporária usada por programas para armazenar arquivos temporários.
C:\ProgramData\
Pasta oculta que contém dados de aplicativos que são compartilhados entre todos os usuários.

O Windows possui outras pastas, mas de início essas são as que mais nos interessam.

$Bibliotecas de Ligação Dinâmica

As DLLs (Dynamic Link Libraries, ou Bibliotecas de Ligação Dinâmica) são arquivos que contêm código, dados e recursos que podem ser utilizados por múltiplos programas no Windows. Elas permitem que programas compartilhem funcionalidades sem precisar duplicar o código em cada aplicação, economizando espaço e facilitando a atualização e manutenção do sistema, e podem ser estruturadas da seguinte forma:

Funções%e%sub-rotinas/
usadas para realização de tarefas específicas.
Variáveis globais/
Dados que podem ser usados por outros programas.
Recursos/
Como ícones, menus, imagens, sons, etc.

Alguns exemplos comuns de dll's são:

kernel32.dll/
Contém funções essenciais do sistema operacional, como gestão de memória e processos.
user32.dll/
Lida com elementos da interface do usuário, como janelas e controles gráficos.
gdi32.dll/
Lida com gráficos e impressão.

As DLLs ajudam a manter o sistema Windows mais eficiente e organizado, evitando a duplicação de código e permitindo que várias aplicações compartilhem recursos comuns.
