#Sistemas Operacionais[Linux]

$O que é o Linux?

Diferente dos sistemas anteriores, Linux não é de fato um sistema operacional, mas um kernel, criado por Linus Torvalds, de código aberto, ou seja, qualquer pessoa pode modificar, usar e distribuir, mas claro que existem restrições que trataremos mais tarde.

O Linux é projetado para ser um sistema multitarefa, ou seja, pode executar várias tarefas ao mesmo tempo. Também é multiusuário, permitindo que várias pessoas usem o sistema simultaneamente com contas separadas.

Existem várias versões do Linux chamadas de distribuições ou "distros", como Ubuntu, Fedora, Debian e CentOS. Cada distribuição pode ser personalizada para diferentes usos, como servidores, desktops ou dispositivos móveis

O Linux é amplamente utilizado em servidores, supercomputadores, dispositivos embarcados e até mesmo em desktops e laptops. Ele é popular em ambientes de TI por sua segurança, estabilidade e flexibilidade. Além disso, o Android, que é um dos sistemas operacionais móveis mais populares, é baseado no kernel Linux.

$Organização

A forma com que os sistemas baseados em Linux organizam seus diretórios é bastante semelhante ao sistema MacOs, pois a base de ambos os sistemas é o Unix.

/(raiz)
O diretório raiz é o ponto de partida do sistema de arquivos. Todos os outros diretórios e arquivos estão organizados sob ele.
/bin/
Contém executáveis binários essenciais, ou seja, comandos básicos usados pelo sistema e pelos usuários, como ls, cat, cp.
/boot/
Armazena arquivos necessários para inicializar o sistema, incluindo o kernel do Linux e o carregador de inicialização (bootloader), como o grub.
/dev/
Contém arquivos especiais que representam dispositivos do sistema, como discos rígidos (/dev/sda), terminais e dispositivos USB.
/etc/
Armazena arquivos de configuração do sistema. Aqui ficam arquivos como /etc/passwd (para usuários) e /etc/hosts (para configuração de rede).
/home/
Diretório de usuários comuns, onde ficam armazenados os arquivos pessoais de cada usuário. Por exemplo, o diretório de um usuário chamado "jose" seria /home/jose.
/lib/
Contém bibliotecas compartilhadas essenciais para os programas localizados em /bin e /sbin, bem como módulos do kernel.
/media/
Usado para montar automaticamente dispositivos removíveis, como pendrives, CDs e DVDs.
/mnt/
Um ponto de montagem temporária para sistemas de arquivos. Administradores podem usar este diretório para montar sistemas de arquivos manualmente.
/opt/
Usado para instalação de pacotes de software de terceiros que não fazem parte da instalação padrão do sistema.
/proc/
Um sistema de arquivos virtual que fornece informações sobre processos em execução e outros dados do kernel em tempo real. Cada processo tem um diretório numerado em /proc.
/root/
Diretório pessoal do superusuário (root). Diferente do /home, é específico para o administrador do sistema.
/run/
Contém arquivos de runtime, que são usados para armazenar dados temporários que o sistema precisa acessar durante a inicialização ou execução.
/sbin/
Contém executáveis de sistema usados principalmente para administração e manutenção do sistema, como fsck, reboot, etc.
/srv/
Armazena dados específicos de serviços fornecidos pelo sistema, como servidores web ou FTP.
/tmp/
Diretório para armazenamento de arquivos temporários. Qualquer usuário pode gravar arquivos aqui, mas eles são excluídos após o reinício do sistema ou em intervalos regulares.
/usr/
Contém arquivos de programas de uso geral do sistema, incluindo bibliotecas, executáveis e documentação.
/var/
Armazena arquivos que variam com o tempo, como logs de sistema, filas de impressão e cache. Por exemplo: /var/log, que armazena logs do sistema e /var/spool, que armazena tarefas pendentes, como trabalho de impressão.

$Linux.

Como o linux não é de fato o sistema operacional, mas sim o kernel, as distribuições que usam esse kernel possuem similaridades gritantes, entre elas, a dependência do terminal. O terminal do linux é a ferramenta mais poderosa que o sistema possui. É através dele que o usuário realiza as principais operações na máquina, a exemplo da instalação de programas.

As distribuições possuem um programa responsável pela instalação, remoção e atualização de outros programas, esses são os gerenciadores de pacotes. Em distros baseadas no linux Debian, como Ubuntu e Kali linux, o gerenciador de pacores usado é o apt. Já em distros baseadas no Arch Linux o gerenciador usado é o packman.

Existem diversas distribuições de sistemas baseados nesse kernel, e por se tratar de um tipo de sistema amplamente utilizado, a partir de agora nós iremos tratar apenas dele no decorrer do conteúdo, principalmente porque existem muito mais ferramentas para linux que para outros sistemas como Windows e MacOs.
