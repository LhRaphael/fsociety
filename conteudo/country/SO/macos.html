#Sistemas Operacionais[MacOs]

$Entendendo um pouco melhor²

O macOS é o sistema operacional desenvolvido pela Apple para seus computadores, como o MacBook, iMac e Mac Pro. Ele é conhecido por sua interface gráfica amigável, alto nível de integração com o hardware da Apple e foco em segurança e privacidade.

Ele é baseado no Unix, o que oferece uma robusta estrutura de segurança e estabilidade. Isso também permite compatibilidade com comandos e ferramentas do Unix.

Um dos pontos fortes do macOS é sua integração com outros dispositivos Apple, como iPhones, iPads, Apple Watch e Apple TV. Funções como Handoff, Continuity e AirDrop permitem uma transição contínua entre dispositivos.

Como o sistema é otimizado para o hardware específico da Apple, ele é reconhecido por seu desempenho estável e alta eficiência energética, principalmente nos Macs com chips Apple Silicon (como o M1 e M2).

$Estrutura do sistema

O MacOs é um sistema baseado em uma arquitetura modular, que significa que possui diversos componentes que funcionam de forma independente e intercambiáveis. Em resumo, a arquiteura do MacOs é definida da seguinte forma:

Hardware/
os componentes físicos dos computadores da Apple, como processadores, memória, SSDs, GPUs, e outros dispositivos periféricos. A Apple projeta seu hardware especificamente para se integrar ao macOS, especialmente com a introdução dos chips Apple Silicon.
Kernel/
O XNU ("X is Not Unix") é o kernel do macOS e está no centro do sistema operacional. Ele gerencia interações básicas entre hardware e software, como gerenciamento de processos, memória, threads e drivers de dispositivos.
Sistema%de%Arquivos/
O Apple File System (APFS) é o sistema de arquivos moderno utilizado no macOS. Ele oferece maior segurança, desempenho, e suporte a recursos como criptografia, instantâneos (snapshots), clonagem de arquivos e pastas e melhor gerenciamento de discos SSD.
Frameworks%e%Bibliotecas/
O macOS oferece diversas bibliotecas e frameworks que suportam o desenvolvimento de aplicativos e serviços.
Camada%de%Interface%Gráfica/
O macOS possui uma interface gráfica centrada no Aqua, o nome da interface de usuário que define o estilo visual e a interação do sistema.
Camada%de%Aplicativos/
O macOS é entregue com uma série de aplicativos nativos, como o Safari, Mail, Calendário, e a suíte de produtividade iWork (Pages, Keynote, Numbers).
Segurança/
O sistema possui uma camada focada apenas na proteção dos dados do usuário, essa camada está acima da camada de aplicativos pois ela é composta por aplicações de segurança.
Virtualização%e%Contêineres/
O macOS suporta virtualização, permitindo que os usuários executem outros sistemas operacionais dentro do macOS. Com a tecnologia Hypervisor, os desenvolvedores podem executar e testar diferentes ambientes em contêineres ou máquinas virtuais.

$Aprofundando

O kernel desse sistema não é um monolito como outros, ele possui elementos que trabalham de forma independente, o Mach, que lida com operações de baixo nível, como gerenciamento de memória e threads, o BSD, responsável por aspectos como o sistema de arquivos, rede e segurança e o I/O Kit, que fornece a interface para gerenciar drivers de dispositivos e interagir com hardware externo (como impressoras, câmeras e dispositivos USB).

Além disso, existem diversos outros componentes no sistema, como Core Service, que fornece APIs para funcionalidades de nível de sistema, como gerenciamento de arquivos, manipulação de dados e persistência de dados; e bibliotecas como Cocoa, Um conjunto de frameworks usado para construir interfaces gráficas no macOS.

Tratando da parte de segurança do sistema, alguns elementos que o sistema possui são:

Gatekeeper/
Impede que o usuário instale ou execute aplicativos que não tenham sido assinados e verificados pela Apple.
FileVault/
Oferece criptografia completa do disco para proteger os dados do usuário.
Sandboxing/
Isola os aplicativos uns dos outros para minimizar os danos que um app pode causar ao sistema ou a outros apps.
XProtect/
Proteção contra malware integrada ao macOS, atualizada regularmente para defender contra novas ameaças.
System%Integrity%Protection/
Impede que softwares maliciosos modifiquem arquivos e pastas cruciais do sistema.

$Diretórios

Os diretórios são distribuidos de forma semelhante aos diretórios dos Unix

/Applications/
Contém todos os aplicativos instalados no macOS. É onde os usuários geralmente colocam aplicativos baixados ou instalados manualmente, assim como muitos dos aplicativos pré-instalados.
/Library/
Esta pasta contém arquivos de suporte do sistema e de aplicativos, como fontes, preferências, extensões e bibliotecas necessárias para o funcionamento do macOS e de seus programas.
/System/
Contém os principais arquivos e componentes do sistema operacional macOS, como o kernel, drivers e outros elementos essenciais para o funcionamento do sistema
/Users/
Esta pasta contém todas as contas de usuários do sistema. Cada usuário tem uma subpasta aqui com seu próprio diretório pessoal.
/bin/
Esta pasta contém executáveis de programas essenciais e comandos usados no terminal do macOS. Exemplos incluem comandos como bash, cat, ls, e outros comandos comuns de linha de comando Unix.
/sbin/
Similar à pasta /bin, mas contém programas e comandos de sistema que só podem ser executados por um usuário com privilégios administrativos, como o sudo.
/usr/
Contém arquivos, bibliotecas e programas do usuário (mas geralmente relacionados ao sistema), como aplicativos Unix que não são parte essencial do macOS.
/private/
Contém arquivos temporários, logs do sistema e outras informações privadas do sistema que geralmente não são acessadas diretamente pelo usuário.
/Volumes/
A pasta onde o macOS monta dispositivos de armazenamento, como discos externos, unidades USB e volumes de rede. Quando você conecta um drive USB ou disco externo, ele aparece como uma subpasta em /Volumes.
/dev/
Contém arquivos de dispositivos (device files), que são interfaces para comunicação com dispositivos de hardware, como discos rígidos, teclados, etc.
/tmp/
Um diretório temporário que o macOS usa para armazenar arquivos que só precisam existir durante a execução de um programa. O conteúdo desta pasta geralmente é apagado automaticamente pelo sistema após reinicializações.
/var/
Contém arquivos variáveis, como logs do sistema e dados que mudam frequentemente. Algumas subpastas importantes: /var/log, que armazena arquivos de log do sistema e aplicativos e /var/mail, que serve para armazenamento de e-mails locais, caso o sistema esteja configurado para tal.

Por hora, isso é mais que suficiente para entendermos o sistema MacOs.
