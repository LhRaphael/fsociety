#Sistemas Operacionais[~]

$Introdução

Após uma introdução ao conteúdo de redes e protocolos web, é hora de entender melhor o que são os sistemas operacionais em sua totalidade. Não é possível explorar vulnerabilidades sem antes saber de quem são as vulnerabilidades que estamos tentando explorar. Dito isto vamos definir o que é um S.O e explorar cada uma de suas camadas.

$O que é?

Um sistema operacional não é um único software mas sim um conjunto de aplicativos que, ao funcionarem em sincronia, são capazes de gerenciar os processos que ocorrem na máquina, coordenar o uso da memória, controlar os dispositivos de hardware, controlar os dados armazenados, dando a possibilidade para que novos dados, como arquivos ou pastas, sejam criados, dados existentes sejam manipulados ou deletados.

Ademais, um sistema operacional também se trata de um conceito abstrato, e como todos os conceitos abstratos ele também possui camadas, essas que estão sobrepostas umas sobre as outras.

$Camadas de um S.O

Aqui vamos a algumas pequenas definições das camadas de um S.O, iremos nos aprofundar bem mais em cada uma nas próximas páginas.

Hardware/
Essa é a camada mais baixa do sistema, nela estão os componentes físicos como processador, pente de memória, placa mãe etc...
Kernel/
O kernel é o núcleo do sistema. Ele gerencia os recursos do sistema, como CPU, memória, dispositivos de entrada e saida etc... Ou seja, o kernel é a camada que cuida da comunicação entre o hardware e o software da máquina.
System%Call/
Nessa camada estão as interfaces fornecidas pelo kernel para que os programas do usuário possam solicitar os demais usos existentes no kernel para executar tarefas.
Aplicação/
Aqui é onde os programas que o usuário utiliza estão localizados, como navegadores, editores de texto, jogos, etc...

$Sistemas vigentes

Atualmente existem 3 sistemas operacionais realmente populares no mercado, na verdade o terceiro não é um sistema em si. Esses são Windows, MacOs e Linux, que será o mais explorado durante o conteúdo que virá.
